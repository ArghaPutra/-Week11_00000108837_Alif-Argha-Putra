# Supabase Setup

This app expects:

- A public storage bucket named `photos`
- A table named `camera_captures`
- Insert/select access for the `anon` role during this assignment demo

## 1. Create the `photos` bucket

In Supabase Dashboard:

- Open `Storage`
- Create a new bucket named `photos`
- Set it to `Public`

## 2. Create the `camera_captures` table

Run this in the SQL editor:

```sql
create table if not exists public.camera_captures (
  id bigint generated by default as identity primary key,
  image_path text not null,
  image_url text not null,
  latitude double precision not null,
  longitude double precision not null,
  captured_at timestamptz not null default now()
);

alter table public.camera_captures enable row level security;
```

## 3. Add table policies

For this coursework-only flow, allow mobile clients to read and insert rows:

```sql
create policy "anon can read camera captures"
on public.camera_captures
for select
to anon
using (true);

create policy "anon can insert camera captures"
on public.camera_captures
for insert
to anon
with check (true);
```

## 4. Add storage policies

Run this in the SQL editor so the app can upload images and read them back:

```sql
create policy "anon can read photos bucket"
on storage.objects
for select
to anon
using (bucket_id = 'photos');

create policy "anon can upload photos bucket"
on storage.objects
for insert
to anon
with check (bucket_id = 'photos');
```

## 5. Environment variables

Your `.env` should contain:

```env
EXPO_PUBLIC_SUPABASE_URL=your_project_url
EXPO_PUBLIC_SUPABASE_ANON_KEY=your_publishable_or_anon_key
```
